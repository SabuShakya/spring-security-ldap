### SETUP ###
Add dependencies:
- unboundId -> open source implementation of ldap server. Will set up a local instance of LDAP.
- spring-ldap-core -> string integration library that works with ldap.
- spring-security-ldap -> helps spring security integrate with ldap.

```xml
    <dependencies>
        <dependency>
            <groupId>com.unboundid</groupId>
            <artifactId>unboundid-ldapsdk</artifactId>
        </dependency>
        <dependency>
            <groupId>org.springframework.ldap</groupId>
            <artifactId>spring-ldap-core</artifactId>
        </dependency>
        <dependency>
            <groupId>org.springframework.security</groupId>
            <artifactId>spring-security-ldap</artifactId>
        </dependency>
    </dependencies>
```

Next we need to add some configurations in our properties file for our local ldap server instance.
Also add a ldap-data.ldif file for users data.

```properties
# PORT WHERE OUR LOCAL EMBEDDED INSTANCE OF LDAP NEEDS TO RUN
spring.ldap.embedded.port=8389
# REFERENCE TO THE FILE WHICH CONTAINS THE SEEDED DATA
# ldif -> LDAP data interchange format, equivalent to sql file
spring.ldap.embedded.ldif=classpath:ldap-data.ldif
# tells what the root node is. it should be read the opposite way starting from end
# eg. org at the top then springframework and everything else is gonna ne below.
# org and springframework doesn't carry any significance as long as the root nodes are same in .ldif file. Can be anything else
spring.ldap.embedded.base-dn=dc=springframework,dc=org
```
